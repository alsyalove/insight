Patterns

Pola standar untuk menangkap pemikiran mentah, log sistem, dan arsitektur data sebelum diekspor ke dalam Vault.

"Penangkapan data yang baik memisahkan observasi dari interpretasi. Biarkan data mentah masuk, interpretasi menyusul."

Prinsip Penangkapan

Frictionless Entry

Proses mencatat harus secepat kilat. Jangan memikirkan hierarki folder atau struktur saat ide baru muncul.

Epistemic Separation

Pisahkan antara fakta (log, data mentah, referensi) dengan asumsi atau pemikiran pribadi.

Saran Implementasi

Gunakan satu inbox terpusat (bisa berupa direktori sementara) untuk semua tangkapan awal sebelum dikategorisasi ke dalam arsitektur sistem atau folder final.

Standarisasi Metadata (Frontmatter)

Gunakan header YAML berikut untuk setiap catatan baru agar kompatibel dengan sistem pengelolaan pengetahuan yang berbasis Markdown.

--- id: YYYYMMDDHHMMSS title: "Judul Singkat" aliases: [] tags: [raw, idea, arsitektur] date: YYYY-MM-DD status: seed --- # Observasi - # Konteks / Logika -

Prosedur Konversi Data
  • 1

    Raw Capture (Input Mentah)

    Tangkap log terminal, output model AI, atau coretan pemikiran tanpa filter pemformatan.

  • 2

    Pembersihan Noise

    Hapus kunci JSON yang tidak relevan (seperti reasoning_content) jika mengekspor riwayat percakapan. Sisakan hanya data esensial.

  • 3

    Injeksi Metadata

    Tambahkan YAML Frontmatter dan simpan dengan ekstensi .md untuk diarsip ke dalam Vault lokal.
